# GUIDE DE CONFIGURATION SUPABASE

## 🚀 Étape 1: Créer un Projet Supabase

1. Allez sur https://app.supabase.com
2. Cliquez sur "New project"
3. Remplissez les informations:
   - **Name:** DataIngest
   - **Database Password:** Créez un mot de passe fort
   - **Region:** Choisissez votre région (ex: Europe - Frankfurt)
4. Attendez que le projet se crée (5-10 minutes)

## 🔑 Étape 2: Récupérer les Credentials

### 2.1 URL du Projet
1. Allez dans **Settings** (engrenage en bas à gauche)
2. Cliquez sur **API**
3. Copiez l'**URL du Project** (format: `https://xxxxx.supabase.co`)
4. Collez-la dans `.env.local`: `VITE_SUPABASE_URL=`

### 2.2 Clé d'API (Anon Key)
1. Dans la même page **Settings > API**
2. Sous **Project API keys**, trouvez la clé **anon (public)**
3. Copiez-la
4. Collez-la dans `.env.local`: `VITE_SUPABASE_ANON_KEY=`

## 📊 Étape 3: Créer la Table Users

1. Allez dans **SQL Editor** (gauche)
2. Cliquez sur **New query**
3. Collez ce SQL:

```sql
-- Créer la table users
CREATE TABLE IF NOT EXISTS users (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  username VARCHAR UNIQUE NOT NULL,
  email VARCHAR UNIQUE NOT NULL,
  name VARCHAR NOT NULL,
  role VARCHAR(20) DEFAULT 'data_entry',
  avatar VARCHAR,
  password_hash VARCHAR NOT NULL,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

-- Créer les indexes
CREATE UNIQUE INDEX IF NOT EXISTS users_email_idx ON users(email);
CREATE UNIQUE INDEX IF NOT EXISTS users_username_idx ON users(username);

-- Insérer les utilisateurs de test
-- Utilisez bcrypt ou crypt() pour hasher les mots de passe
-- Temporairement, nous utilisons du texte en clair (⚠️ NE PAS FAIRE EN PRODUCTION)

INSERT INTO users (username, email, name, role, password_hash) VALUES
('admin', 'admin@mentora.com', 'Fatima Admin', 'admin', 'admin123'),
('youssef', 'youssef@mentora.com', 'Youssef Data Entry', 'data_entry', 'user123'),
('amina', 'amina@mentora.com', 'Amina Data Entry', 'data_entry', 'user123'),
('karim', 'karim@mentora.com', 'Karim Data Entry', 'data_entry', 'user123'),
('salma', 'salma@mentora.com', 'Salma Data Entry', 'data_entry', 'user123')
ON CONFLICT (email) DO NOTHING;
```

4. Cliquez sur **RUN**
5. Vérifiez: Allez dans **Table Editor** > **users** - vous devriez voir les 5 utilisateurs

## ⚙️ Étape 4: Créer la Fonction RPC verify_password

1. Allez dans **SQL Editor**
2. Créez une nouvelle requête:

```sql
-- Créer la fonction verify_password
CREATE OR REPLACE FUNCTION verify_password(p_email TEXT, p_password TEXT)
RETURNS BOOLEAN AS $$
DECLARE
  stored_hash VARCHAR;
BEGIN
  SELECT password_hash INTO stored_hash
  FROM users
  WHERE email = p_email;
  
  -- Comparaison simple (EN PRODUCTION, utilisez bcrypt!)
  RETURN stored_hash = p_password;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Donner l'accès au public (anonymous users)
GRANT EXECUTE ON FUNCTION verify_password TO anon, authenticated;
```

3. Cliquez sur **RUN**

## 🪣 Étape 5: Créer les Buckets de Stockage

1. Allez dans **Storage** (gauche)
2. Cliquez sur **Create bucket**
3. Nommez-le: `documents`
4. Une fois créé, cliquez dessus
5. Cliquez sur **+ New folder**
   - Créez `pdfs`
   - Créez `json`
6. Pour chaque dossier, cliquez et **Setup RLS** (Row Level Security)
7. Rendez-les **public** (lisibles par tous les authenticated users)

## ✅ Étape 6: Vérifier la Configuration

Vérifiez que votre `.env.local` contient:
```
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGci...
```

## 🧪 Étape 7: Tester

```bash

```

Allez sur http://localhost:3000, vous devriez voir la LoginPage.

Testez avec:
- Email: `admin@mentora.com`
- Mot de passe: `admin123`

Si ça ne marche pas, vérifiez les points suivants.

## 🐛 Troubleshooting

### Erreur 400 (Bad Request)
**Cause:** Variables d'environnement vides ou invalides
**Solution:** Vérifiez que `.env.local` a les bonnes valeurs de `VITE_SUPABASE_URL` et `VITE_SUPABASE_ANON_KEY`

### Erreur "Email incorrect"
**Cause:** L'utilisateur n'existe pas dans la table `users`
**Solution:** Assurez-vous que vous avez inséré les utilisateurs de test avec le SQL ci-dessus

### Erreur "Mot de passe incorrect"
**Cause:** La fonction RPC `verify_password` ne retourne pas true
**Solution:** Vérifiez que la fonction RPC a été créée correctement et que vous utilisez un mot de passe valide

### Erreur CORS
**Cause:** Supabase bloque la requête de votre origin
**Solution:** Allez dans **Settings > API > CORS** et ajoutez `http://localhost:3000`

### TypeError: supabase is undefined
**Cause:** Le client Supabase n'a pas pu être créé (credentials manquantes)
**Solution:** Vérifiez `.env.local` et relancez `npm run dev`

## 🔐 Production Security (Important!)

⚠️ **EN PRODUCTION:**
1. **Utilisez bcrypt ou Argon2** pour hasher les mots de passe (pas du texte en clair!)
2. **Activez le RLS** (Row Level Security) sur la table `users`
3. **N'exposez jamais** votre Service Role Key (gardez-la côté serveur uniquement)
4. **Utilisez HTTPS** partout
5. **Activez l'authentification 2FA**
6. **Limitez les requêtes REST** avec un rate limiter
